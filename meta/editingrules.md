# Core Content Rules
1. Utility Above All: The primary goal is practical value for both beginners and experts. Technical tutorials, compatibility specs, and execution guides will always take priority over historical trivia or brand background.

2. Format & Layout: All new pages must be written in Markdown. For sidebars and custom widgets, use `<div class="infobox">...</div>` with custom CSS where needed.

3. Pragmatic Knowledge: Sources are highly encouraged, but don't let the lack of a citation block useful knowledge. If information is helpful to riders, document it first; source it later.

4. Source Priority: When conflicts arise, sourced information strictly supersedes unsourced claims or community hearsay.

5. AI Usage: Using AI tools (LLMs) as writing assistants for formatting or grammar is fine, but you shouldent use them for researchsince LLMs lack knowledge on freeskating. Contributors bear full responsibility for any content they submit.
